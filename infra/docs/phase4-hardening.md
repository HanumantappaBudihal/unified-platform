## Remaining (needs a Kubernetes cluster / cloud — manifests + plan)

These are infra-level HA/DR items from [production-readiness-gaps.md](production-readiness-gaps.md); the
[Crossplane spike](../crossplane/spike/) proved the declarative path for the data tier.

- **PostgreSQL HA** — replace single Postgres with **CloudNativePG** (primary + replicas, automated failover, continuous WAL archiving / PITR). The registry DB and per-tenant DBs both move to CNPG clusters.
- **Kafka HA** — 3 brokers, `replication.factor=3`, `min.insync.replicas=2`, `unclean.leader.election=false` (Strimzi).
- **Zero-downtime** — readiness/liveness probes + PodDisruptionBudgets + rolling updates on every Deployment; HPA for the stateless tiers.
- **Encryption** — TLS everywhere via cert-manager (the self-signed `certs/` are generated by `certs/scripts/generate-certs.sh` but were never run — Keycloak HTTPS is currently disabled for that reason); encryption-at-rest via the storage class / KMS.
- **Network policies** — default-deny `NetworkPolicy` per tenant namespace (files staged under `infra/k8s/network-policies/`).
- **DR drills** — schedule the restore drill above against real backups; record RTO/RPO.
- **Access reviews / pen-test / policies** — process controls for the SOC2 audit (out of band).
